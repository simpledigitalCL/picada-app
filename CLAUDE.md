# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project overview

Picada App is a gastronomic discovery mobile/web app for the Chilean market. It's a Next.js 15 (App Router) SPA wrapped in Capacitor for Android/iOS, backed by Supabase (PostgreSQL + Auth + Storage), with Claude AI for dish scanning and Leaflet for interactive maps.

## Commands

```bash
npm run dev              # Dev server on port 3002 (not 3000)
npm run build            # Production build
npm run lint             # ESLint

# Android / Capacitor
npm run cap:copy         # Copy build to Android project
npm run cap:sync         # Sync Capacitor plugins + assets
npm run cap:open:android # Open Android Studio
npm run android:sync:emulator  # Point to http://10.0.2.2:3002 (emulator host)
npm run android:sync:prod      # Point to production domain
```

## Architecture

The app is a **tab-based SPA** — navigation is driven by React state in `app/page.tsx`, not by Next.js Router. This makes it work as a PWA and inside the Capacitor WebView without complex browser history.

```
app/page.tsx       ← SPA root: manages tabs, auth, modals, gamification init
app/api/**         ← Server-side API Routes (all use Supabase service role)
components/        ← UI grouped by domain (see below)
lib/               ← Business logic grouped by domain (see below)
supabase/          ← schema.sql (idempotent) + migrations/ (apply in date order)
feedie/            ← Standalone prototype; NOT part of the main build
android/           ← Generated by Capacitor; do not edit manually
```

### Supabase clients

- Use `lib/supabase.ts` in browser/client components.
- Use `lib/supabase-server.ts` in API Routes — uses service role key to bypass RLS when needed.
- All RLS policies are in `supabase/schema.sql`. Public reads on `places` and `posts`; writes require `auth.uid() = user_id`.

### API Routes pattern

Each route lives in `app/api/<domain>/route.ts` and exports named `GET`/`POST` handlers. They use the admin Supabase client. Zod schemas for validation live in `lib/validation/`.

### Tag system (core of personalization)

Tags connect users to places via affinity scores. Three layers:
1. **Classification** (`lib/places/category-filter.ts`) — raw Google Maps categories → `local_slug`
2. **Suggestion scoring** — curated base tags + community frequency (`tag_relations_stats`) + user affinity (`user_tag_affinity`), combined via Supabase RPC at `/api/suggestions`
3. **Affinity accumulation** — `user_tag_affinity` updated by reviews (+3), visits (+2), likes (+1), searches (+1), profile setup (+5)

### Gamification

- XP events are emitted client-side via `lib/gamification/events.ts` and persisted server-side via `/api/events`.
- Achievement engine (`lib/gamification/achievement-engine.ts`) triggers toast notifications via `components/gamification/AchievementToast.tsx`.
- Leaderboard served by `/api/leaderboard`.

### AI integrations

- **Dish scanner** (`/api/scanner`): receives base64 image, calls Anthropic Claude Vision, returns dish name, nutrition estimate, and suggested tags.
- **Auto-tagging** (`lib/places/auto-tagging.ts`): triggered in background when a new place is registered; runs inference via `lib/inference/tag-processor.ts`.

## Conventions

- **Package manager**: npm (only `package-lock.json` — do not use pnpm or yarn).
- **Brand color**: `orange-500` (#f97316) — active tabs, primary actions.
- **HTML sanitization**: all user-generated HTML passes through `isomorphic-dompurify` (`lib/utils/sanitize.ts`) before rendering.
- **Anonymous identity**: pre-login users get a localStorage identity (`lib/auth/identity.ts`); affinity and history migrate to the real profile on login.
- **Capacitor note**: `next.config.mjs` has `images.unoptimized: true` — required for images inside the Android WebView.
- **Mobile strategy**: Next.js + Capacitor only. No React Native / Flutter migration planned. iOS support via `npx cap add ios` when needed.
- `feedie/` is an isolated prototype with its own `package.json` — not imported by the main app.

## Database

Apply schema to a Supabase project via the SQL Editor:
1. Run `supabase/schema.sql` (idempotent).
2. Run each file in `supabase/migrations/` in chronological order.

Required PostgreSQL extensions: `pgcrypto`, `pg_trgm`.
